# M9 접근성 감사 대본 — TalkBack / VoiceOver 실주행 (2026-07-05)

> **성격: `[AI→사람]` 지그.** `[AI]` 정적 리포트(§3-5 대비비·contentDescription 커버리지)를 근거로 한 실기기 감사 대본.
> AI가 대본을 깔고, 사람은 대본대로 1회 주행·컨펌한다. 정본 테스트: `AccessibilityContrastTest`(대비비)·
> `ContentDescriptionCoverageTest`(커버리지). `[사람]` 잔여 = TalkBack/VoiceOver **실주행**·Dynamic Type 실반영.

## A. `[AI]` 정적 선검출 결과 (실기기 전에 이미 닫힌 것)

### A-1. WCAG 대비비 — **전 쌍 AA 통과** (게이트 green)
`AppColors` 11토큰(라이트/다크)의 전경(text·textDim·textMuted·accent·accent2·accentAI) × 배경(bg·surface·surface2)
18쌍 × 2스킴 = 36쌍 모두 **AA(4.5:1) 이상**. 최저값 = `accentAI/surface2`(라이트) **4.62:1**. 대부분 AAA(7:1↑).

| 스킴 | 최저 대비 쌍 | 비율 | 등급 |
|---|---|---|---|
| DARK | textMuted/surface2 | 5.04 | AA |
| LIGHT | accentAI/surface2 | 4.62 | AA |

→ **토큰 조정 불필요.** 실기기 감사는 "코드 대비비"가 아니라 **실 디스플레이/밝기/색보정에서의 체감**만 확인.

### A-2. contentDescription 커버리지 — 그래픽 위젯 **0개**
UI가 **전면 텍스트/이모지 기반**(Compose `Icon`/`Image`/`painterResource`/vector/Canvas **0개**, semantics 0개).
→ 그래픽 `contentDescription` 누락은 구조적으로 없음. **대신 이모지-only Text가 인터랙티브일 때** OS 이모지
TTS(스크린리더가 이모지를 유니코드 이름으로 읽음)에 의존 → 아래 B에서 실주행으로만 판정 가능한 잔여.

## B. `[사람]` 실주행 대본 — TalkBack(Android) / VoiceOver(iOS)

각 화면을 스크린리더 켠 채 **스와이프 순서대로** 훑으며, 읽히는 라벨이 **의미를 전달하는지** 확인.

### B-1. 검색 화면
1. 검색 입력창 → "검색어 입력" 류 힌트가 읽히는가 (TextField placeholder)
2. 검색 결과 카드 스와이프 → 용어명·카테고리 배지가 순서대로 읽히는가
3. **카테고리 배지(이모지 포함 시)** → 이모지가 뜻 없는 이름으로만 읽히면 ☐ 기록 (라벨 보강 후보)
4. 최근 검색 칩(FlowChip) → 칩 텍스트가 읽히고 탭 가능으로 인식되는가

### B-2. 상세 화면
5. 뒤로("← 뒤로") → "뒤로"가 읽히는가 (← 글리프 단독이 아니라 텍스트 동반 확인)
6. 북마크 토글 → **상태(북마크됨/안 됨)**가 읽히는가. ⚠️ 아이콘/이모지만이면 상태 불명 → ☐ 기록
7. 어원·명명 이유 본문 → 순서대로 낭독, 건너뜀 없는가
8. 메일/공유/복사/평가 액션 → 각 버튼 목적이 라벨로 전달되는가

### B-3. 북마크·히스토리
9. 빈 상태(EmptyState) → 안내 텍스트가 읽히는가 (이모지 헤더는 장식 — 뒤 텍스트가 의미 전달)
10. 목록 항목 스와이프 순서 자연스러운가, 삭제/탭 동작 인식되는가

### B-4. 설정·온보딩·라이선스
11. 설정 토글(외관 3모드·데이터 수집) → 현재 값과 역할이 읽히는가
12. 외관 전환 → 스크린리더 켠 채 라이트/다크 전환이 즉시 반영·낭독 방해 없는가
13. 온보딩(최초 1회) → 페이지 순서·완료 버튼 읽히는가
14. 라이선스 화면 → 긴 OFL 텍스트 스크롤이 스크린리더로 탐색 가능한가

### B-5. Dynamic Type / 글꼴 크기
15. OS 글꼴 크기를 최대로 → 레이아웃 깨짐(잘림·겹침) 없는가, 하이브리드 폰트(한글 시스템·영문 DM) 정상 스케일

## C. 컨펌 체크 (사람)
- ☐ B-1~B-5 전 화면 스크린리더 1회 주행 완료
- ☐ 이모지-only 인터랙티브(B-3·B-6 북마크 상태) 라벨 적절성 판정 — 부족하면 `Modifier.semantics { contentDescription = ... }` 보강 백로그
- ☐ Dynamic Type 최대에서 레이아웃 무손상
- ☐ 대비비는 A-1로 선검증됨 — 실기기 체감만 확인

> **잔여 백로그**: 스크린리더가 이모지 상태를 불명확히 읽는 항목이 나오면 `semantics` 보강은 M9 게이트가 아닌
> 후속 개선(릴리즈 비차단). 단, **북마크 on/off 상태 낭독**이 불가하면 게이트 승격 검토(핵심 기능 접근성).
