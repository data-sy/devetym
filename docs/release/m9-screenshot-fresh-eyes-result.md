# App Store 미리보기 백지 설계 — 프레시아이 결과 (2026-07-14)

> 작성 조건: 기존 문안·프레이밍 산출물(`m9-screenshot-capture-handoff.md`, `m9-store-metadata-draft.md`, `ios-launch-decision-prompt.md`, `framed*/`)을 열지 않고, 레포 문서와 원본 스크린샷만으로 처음부터 설계했다.
>
> **개정 R1 (2026-07-14, 사람 검토 반영)**: 컷 2↔3 순서 교체(AI 컷을 앞으로), AI 컷 캡션에서 '사전에 없는 용어' 프레임 제거. 근거: 내장 콘텐츠도 동일 프롬프트로 AI가 미리 생성한 것이라 소비자 관점에서 사전 유무 구분은 무의미하며, 훑고 지나가는 노출 맥락에서는 조건절 없는 직관적 주장("AI가 어원을 설명해준다")이 우선.

## 1. 앱 이해 요약

- **본질**: 개발 용어의 뜻이 아니라 **"왜 그 이름이 붙었는지"(어원 + 작명 이유)** 를 한국어로 설명하는 사전. 상세 화면의 "어원 / 왜 이 이름인가" 2단 구조가 제품 그 자체다 (근거: `docs/product/prd.md` 문제 정의·콘텐츠 구조, `~/devetym-shots/ios/03-term-detail.png`).
- **차별점**: ① 뜻 검색은 어디서나 되지만 작명 이유는 어디에도 없다는 빈틈을 정확히 침 ② 내장 DB라 오프라인 즉답 + 사전에 없는 용어는 AI가 이어받는 폴백 구조 (근거: `README.md` 데이터 흐름, `04-ai-generated.png`의 "AI 생성" 뱃지) ③ 하단 검색창(한 손 도달성)·모노스페이스+세리프 에디토리얼 다크 UI라는 뚜렷한 시각 정체성 (`02-search-suggestions.png`, `09-main-light.png`).
- **타깃**: 한국어 사용 주니어 개발자·직무 전환자 — 강의 중 낯선 용어를 만나는 사람, 면접 준비로 CS 용어를 정리하는 사람 (근거: `docs/product/prd.md` 타겟 유저·유저 스토리 US-01~03).
- **3초 멈춤 가설**: 스토어를 스치는 타깃은 "bug가 진짜 벌레였다고?" 같은 **구체적 어원 한 방**에 멈춘다. 기능 나열보다, 첫 컷에서 실제 용어 카드 하나를 그대로 보여주는 것이 최고의 후킹이다.

## 2. 컷 구성안 (5컷)

검색 결과 노출은 앞 1~3컷뿐이고 순서·연속성이 보장되지 않으므로, **1~3컷에 제품의 존재 이유(가치→AI 차별점→상호작용)를 완결적으로 배치**하고 4~5컷은 리텐션·취향 요소로 뒷받침한다. 모든 캡션은 단독 완결형이다.

### 컷 1 — 핵심 가치: 용어 카드 그 자체
- **화면**: `03-term-detail.png` (bug 상세, 다크) — 기존 원본 재사용
- **캡션**: **개발 용어, '왜 이 이름인지'까지 알려드려요**
- **메시지·배치 근거**: 이 앱이 파는 것은 검색 기능이 아니라 "작명 이유"라는 콘텐츠다. bug 카드는 그레이스 호퍼 일화라는 누구나 아는 이야기로 어원 섹션이 저절로 읽히고, "어원 / 왜 이 이름인가" 레이아웃이 캡션의 증거가 된다. 검색 결과 첫 노출에서 제품 정의를 3초 안에 끝내는 컷.

### 컷 2 — 차별점: AI가 어원을 설명 (R1 개정)
- **화면**: `04-ai-generated.png` (polyfill, "AI 생성" 뱃지 노출, 다크) — 기존 원본 재사용
- **캡션**: **어떤 용어든 물어보세요 — AI가 어원까지 설명해요**
- **메시지·배치 근거**: 자동완성은 모든 사전 앱이 하는 기능이라 차별점 증명이 못 되므로, 노출이 보장되는 두 번째 자리에는 이 앱의 실제 차별점(AI 어원 설명)을 조건절 없이 선언한다. '사전에 없는 용어는'이라는 내부 구현 구분은 소비자에게 무의미 — 내장 콘텐츠도 같은 프롬프트로 AI가 미리 생성한 것이라, 소비자 관점의 사실은 "모든 어원 설명을 AI가 만든다"이다. 화면 속 "AI 생성" 뱃지가 캡션의 증거이자 투명성 표시. "설명해요"는 과정 서술이라 즉답 약속이 아니다.

### 컷 3 — 핵심 상호작용: 검색
- **화면**: `02-search-suggestions.png` ("de" 입력 → 후보 리스트, 다크) — 기존 원본 재사용
- **캡션**: **한 글자씩 좁혀지는 검색 — 내장 사전이라 오프라인도 OK**
- **메시지·배치 근거**: 강의 중 빠른 검색(US-01)이라는 사용 맥락을 보여준다. 자동완성 리스트가 콘텐츠의 폭(deadlock, decorator, dependency-injection…)을 숫자 없이 증명한다. 오프라인 즉답은 내장 DB의 실제 속성(`README.md` 데이터 흐름)이고 AI 경로가 아니므로 즉시성 과장에 해당하지 않는다. 1~3컷만 노출돼도 가치–차별점–사용법이 완결된다.

### 컷 4 — 리텐션: 북마크 복습
- **화면**: `05-bookmarks.png` (북마크 목록, 다크) — 기존 원본 재사용
- **캡션**: **저장해 둔 용어만 모아, 면접 전에 복습**
- **메시지·배치 근거**: 타깃의 두 번째 페르소나(면접 준비생, US-03)를 직격한다. bug·mutex·kernel이 한 줄 요약과 함께 쌓인 목록이 "모아보기"의 실물 증거. 상세 페이지까지 본 사용자에게 재방문 이유를 심는 컷.

### 컷 5 — 손맛·취향: 라이트 테마 + 하단 검색창
- **화면**: `09-main-light.png` (라이트 모드 메인, 하단 검색창·최근 검색 칩) — 기존 원본 재사용
- **캡션**: **검색창은 엄지 닿는 아래쪽 — 라이트 테마도 준비돼 있어요**
- **메시지·배치 근거**: 다크 4컷 뒤에 라이트 화면이 시각적 반전을 만들어 끝까지 스와이프한 사용자의 주의를 회수한다. 하단 검색창은 PRD가 명시한 의도적 설계(한 손 도달성)라 사실 기반이며, 세트 전체가 "다크 전용 앱"으로 오독되는 것도 방지한다.

## 3. 비주얼 방향

- **배경색**: 다크 배경 금지 제약에 맞춰 **웜 아이보리(#F5F2E9 계열) 단색**을 기본으로 한다. 다크 UI 스크린과 명도 대비가 커서 디바이스 경계가 또렷하고, 앱의 라임 액센트(#C9F24B 계열)와 같은 옐로-그린 계열이라 이질감 없이 이어진다. 컷 5(라이트 화면)만 배경을 반 톤 어두운 웜 그레이(#E7E3D8)로 내려 화면과 배경이 뭉개지지 않게 한다. 전 컷 단색 유지 — 그라데이션·패턴 없이 콘텐츠(용어 카드)가 주인공.
- **디바이스 프레임**: 실기기 베젤 프레임(아이폰 다크 티타늄 톤) 사용, 화면 대비 약 80~85% 크기로 중앙 하단 배치해 상단에 캡션 존을 확보. 프레임 뒤에 은은한 드롭 섀도로 아이보리 배경에서 부양감만 주고 기울임·원근은 쓰지 않는다(용어 텍스트 가독이 곧 셀링 포인트라 정면 고정).
- **캡션 타이포**: 상단 좌측 정렬(또는 중앙) 2줄 이내. 본문은 **묵직한 한글 산세리프 볼드(Pretendard Bold류), 잉크 블랙(#1A1A18)** — 아이보리 배경 위 최대 대비. 핵심 구("왜 이 이름인지", "오프라인", "AI", "면접 전") 는 라임 형광펜 스타일 하이라이트(배경 칠) 로 강조해 앱 내부 액센트 컬러와 브랜드를 일치시킨다. 라임을 글자색으로 쓰는 것은 아이보리 위 대비 부족이라 금지, 반드시 배경 칠로만.
- **톤**: 광고 감탄사 없이 평서형·존대 종결("~해요", "~드려요")로 앱 내부 카피 톤과 통일.

## 5. 자가 체크 (안전장치 1~7)

1. **사실 기반**: 위반 없음 — 모든 캡션 주장(작명 이유 설명·AI 어원 설명·자동완성·오프라인 내장 사전·북마크·라이트 테마·하단 검색창)은 PRD/README와 원본 스크린샷에서 직접 확인한 출시 기능만 사용, 알림 등 백로그 기능 언급 없음. "어떤 용어든"(컷 2)은 사전 미수록 용어도 AI 생성 경로가 실제로 처리하므로 사실 기반.
2. **고정 수량 숫자 금지**: 위반 없음 — "650개" 등 어떤 수량도 캡션에 쓰지 않았다("한 글자씩"은 입력 단위 서술이지 콘텐츠 수량이 아님).
3. **즉시성 과장 금지**: 위반 없음 — AI 캡션(컷 2)은 "설명해요"로 능력을 서술할 뿐 "즉시/치는 순간" 같은 즉답 약속이 없다. "오프라인도 OK"(컷 3)는 AI가 아닌 내장 DB 경로의 실제 속성.
4. **다크 배경 금지**: 위반 없음 — 전 컷 웜 아이보리/웜 그레이 라이트 배경, 검은색·앱 동일 다크 톤 미사용.
5. **규격**: 위반 없음 — 6.9" 1320×2868 px 기준 설계 + 6.5" 1242×2688 병행 산출 전제, iPhone 전용·스토어 언어 한국어.
6. **컷별 자기완결**: 위반 없음 — 5개 캡션 모두 앞 컷 참조·연속 서사 없이 단독으로 읽힌다.
7. **표현 규범**: 위반 없음 — 남성형 디폴트 비유·명명 미사용.
